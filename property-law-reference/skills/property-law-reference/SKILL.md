# Property Law Reference Directory

## Purpose

When giving property-related advice or explanations, always ground your response in authoritative sources. This skill maps common topics to the definitive URL and tells you how to cite it.

**Never present training knowledge as the authority.** Always fetch the live page and cite the URL.

## How to use

1. Identify the topic from the user's question
2. Find the matching reference below
3. Fetch the URL using web_fetch
4. Answer using the fetched content, citing the source
5. If no reference matches, use web search targeted at the authoritative domains listed below

## Citation format

Always cite like this:

> According to [Source Name], [statement]. (Source: [URL])

For example:

> According to HMLR Practice Guide 27, a lease extension under the 1993 Act requires the leaseholder to have owned the lease for at least two years. (Source: https://www.gov.uk/government/publications/leasehold-property/practice-guide-27-leasehold-property)

## Integration with Moverly DE

When a Moverly DE flag includes a `legalContext` field with a URL, always cite that URL. The DE's legalContext is the most authoritative source because it's tied to the specific scenario and evidence that triggered the flag.

Priority order for citations:
1. **DE flag legalContext URL** — scenario-specific, highest authority
2. **This reference directory** — topic-level, fetch live content
3. **Targeted web search** — fallback for edge cases

## Important

- **Never give legal advice.** Always frame as information, not recommendation.
- **Always cite the source URL.** No uncited assertions about legal rights or obligations.
- **Flag stale content.** If a fetched page mentions dates or legislation that seems outdated, note this.
- **Suggest professional advice.** End substantive responses with "Consider discussing this with your conveyancer" or equivalent.
